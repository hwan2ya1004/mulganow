# 물가나우 마스터 세션 정기 보고 (2026-07-21)

## 0. 세션 도중 사용자가 직접 발견한 라이브 버그 — 원인 파악 및 수정 완료

사용자가 라이브 사이트(`/blog/2026-07-20-price-watch.html`, 오늘 아침 자동발행 글)를 직접
보고 지적: "닭/육계(kg) 오늘 시세 확인" 섹션에 걸린 애드픽 링크 중 하나가 실제로는
**"[하프클럽/알티피아]ZF 부위 닭고기 벽걸이장식 설명 철판액자"(식품이 아닌 인테리어
장식품)**였고, 가격도 10,820원으로 6,210원대 시세와 무관했음.

**원인 확인**: `/api/shop-prices?q=닭` API를 마스터가 직접 재호출해 동일한 상품이
반환되는 것을 재현. `mulganow/mulganow/backend/app.py`의 `NON_FOOD_KEYWORDS`
(비식품 상품 제외 키워드 목록)에 벽걸이/액자류 키워드가 빠져 있어, 제목에 "닭고기"라는
단어만 들어가면 통과되고 있었음.

**조치**:
- `app.py`의 `NON_FOOD_KEYWORDS`에 "벽걸이", "액자", "인테리어소품" 등 추가 (커밋 `1a3ba5a`).
- 이미 라이브에 배포돼 있던 7/20 자동발행 글의 해당 링크를, API 재조회로 확인한 실제
  닭고기 상품 2건("캠핑닭고기... 닭목살 1kg" 19,770원, "크레잇 간장닭 닭다리살 스테이크
  1.08kg" 31,530원)으로 직접 교체 (커밋 `03abfec`).
- 배포 후 재확인: `curl`로 라이브 페이지에서 "닭고기 벽걸이장식" 문자열이 더 이상 나오지
  않고 "캠핑닭고기"/"크레잇 간장닭"만 남아있음을 직접 확인.

## 1. 콘텐츠 품질 관련 사용자 피드백 — 작성팀 초안 재작업

작성팀이 오늘의 신규 특집("상추·배는 오르고 피망은 뚝")을 완성했으나, 마스터가 직접 읽어본
결과 자동발행 템플릿(도입 한 줄 + 통계박스 + 항목별 뻔한 문장)을 상품만 바꿔 반복한
수준이었음. 사용자도 같은 시점에 "블로그 글 내용이 너무 대충인데 이렇게 써서 사람들이
오겠냐" 라고 직접 지적.

**조치**: 마스터가 본문을 다시 작성 — 상추값 상승은 장마철 잎채소 폐기·출하 회복 시차로
설명하고 보관 팁 추가, 배값 상승은 여름철 저장량 감소라는 계절적 이유와 "추석까지 여유
있으니 급하지 않으면 미뤄도 된다"는 실질적 조언 추가, 피망 하락분은 "상추 대신 피망으로
대체"라는 이번 주 전체를 관통하는 절약 전략으로 마무리(커밋 `ea3f282`). 데이터·가격은
전부 실제 API 응답 그대로 유지, 애드픽 링크 2건(청상추 로메인 1kg SSG 11,900원, 청피망
1kg Hmall 10,200원)은 `/api/shop-prices` 재호출로 실제 매칭임을 직접 재확인.

## 2. 애드픽 전환 0건 관련 — 원인으로 추정되는 정황과 한계

사용자가 애드픽 대시보드(최근 7일 클릭 25 / 전환 0 / 커미션 ₩0)를 보여주며 문제를 제기.
위 0번 항목(비식품 상품 오매칭)이 최소 일부 클릭 낭비의 실제 원인이었을 가능성이 있고,
1번 항목(형식적인 자동 템플릿 카피)도 구매 전환을 끌어내기엔 설득력이 약했을 수 있음.
다만 **사이트가 생긴 지 얼마 안 됐고 트래픽 자체가 적어(GA4 기준 최근 7일 `/blog` 21뷰)
클릭 25건 규모에서 전환 0건이 통계적으로 이례적인 수치인지는 마스터가 판단할 근거가
부족함** — 이 부분은 확인 못함으로 남기고, 위 두 가지 원인은 고쳤으니 다음 점검 주기에
전환율 변화를 지켜볼 필요가 있음.

## 3. 오늘 4개 팀 소집 결과 (마스터 직접 재검증)

**기획팀**: GA4(`properties/544943258`) 재쿼리 결과를 마스터가 재실행해 확인 —
최근 7일 `/blog` 21뷰/7명, 여름 건조케어 글 평균 체류 178.4초(직전 1위였던
summer-fruit-price-drop.html의 152.7초를 추월), Organic Search 채널은 11일째 0건.
`content_calendar_notes.md`에 실측치와 우선순위(중복 7/25는 자동화에 맡기고 수동
소재에서 제외, 위생/뷰티 카테고리 추가 검토)를 갱신 저장한 것을 파일 열람으로 확인
(커밋 `785827c`).

**개발팀**: `git log`로 중복 작업 없음을 먼저 확인 후, `app.py`의 단위 중량 힌트 조회가
KAMIS 원본 이름으로만 조회되어 "참다래(키위)" 등에서 힌트가 누락되던 버그를 발견·수정하고
외부 API 호출 없는 핵심 로직 유닛테스트 50건을 신규 작성. 마스터가 직접
`python -m py_compile app.py test_helpers.py`와 `python -m unittest test_helpers -v`를
재실행해 **50개 전부 통과**함을 확인 (커밋 `1a3ba5a`).

**작성팀**: 1순위(폭염/여름휴가 생필품, 카테고리 다양성)로 참가격 데이터를 뒤졌으나 8%
이상 절약 조건을 만족하는 매칭이 하나도 없어 실패 — 시도한 품목과 실패 사유를 구체적으로
보고받음. 2순위(today-prices month_change_pct)로 성공, 위 1번 항목대로 마스터가 본문을
보강해 발행.

**홍보팀**: 작성팀 신규 글을 기반으로 네이버 원고를 작성했으나 원래 초안(자동템플릿
수준)에 맞춰져 있어, 마스터가 보강된 본문 내용에 맞게 원고도 다시 갱신함
(`naver_drafts/latest.md`, 커밋 `ea3f282`). `curl`로 `/blog`(200)와 오늘 자동발행 글(200)
라이브 반영을 직접 재확인. 페이스북/쓰레드 자동 게시는 여전히 미구현(Meta 토큰 미발급) —
사용자 명시적 승인 없이 시도하지 않음.
