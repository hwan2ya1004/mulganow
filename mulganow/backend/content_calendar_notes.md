## 2. SEASONAL_EVENTS 캘린더 공백 분석

`generate_blog_post.py`의 자동 발행 스케줄은 `.github/workflows/generate-blog-post.yml`의
cron `0 22 * * 1,4`(UTC) — 즉 **매주 화/금요일 07:00 KST**에 발행됩니다(주석은 "월/목"이라
적혀 있지만 UTC→KST 환산하면 실제 발행일은 화/금). 오늘(2026-07-19, 일)부터
말복(8/14)까지 발행 예정일과 절기 이벤트 매칭:

| 발행일(KST) | 활성 절기 이벤트 | 비고 |
|---|---|---|
| 7/21(화) | 중복(7/25, lead 5일 → 7/20~25 활성), 장마(~7/25 활성) | 커버됨 |
| 7/24(금) | 중복, 장마(마지막날) | 커버됨 |
| **7/28(화)** | **없음** | **공백** |
| **7/31(금)** | **없음** | **공백** |
| **8/4(화)** | **없음** | **공백** |
| **8/7(금)** | **없음** | **공백** |
| 8/11(화) | 말복(8/14, lead 5일 → 8/9~14 활성) | 커버됨 |
| 8/14(금) | 말복 당일 | 커버됨 |

**핵심 공백: 7/26(중복 다음날)부터 8/8(말복 활성 시작 전날)까지 약 2주, 자동 발행 4회
분량이 절기 트리거 없이 순수 "등락률 TOP" 글로만 채워짐.** 장마도 7/25 종료, 추석은
9/25로 아직 멀어 이 구간은 SEASONAL_EVENTS 어디에도 걸리지 않는 완전 공백.

**제안**: `SEASONAL_EVENTS`에 "폭염/여름휴가" 이벤트를 하나 추가해 이 공백을 메우는 것을
검토. 예시:
```python
{
    "name": "폭염/여름휴가",
    "start": "2026-07-26",
    "end": "2026-08-08",
    "keywords": ["수박", "참외", "복숭아", "옥수수", "얼음"],  # 농산물(KAMIS) — 현재 API 장애로 당분간 매칭 안 될 가능성 높음
    "consumer_keywords": ["아이스박스", "모기", "벌레퇴치", "자외선차단", "선크림", "휴대용선풍기", "물티슈"],
},
```
농산물 keywords는 KAMIS 복구 후를 대비해 남겨두되, 아래 3번 항목 때문에 당분간은
`consumer_keywords` 쪽이 실질적으로 콘텐츠를 채울 축이 될 것으로 예상.

## 3. KAMIS API 장애 현황 및 자동화 리스크 (중요)

- `/api/today-prices` 직접 호출 재확인: `https://mulganow.vercel.app/api/today-prices?cls=01`
  → HTTP 502, 본문 `{"error":"KAMIS API 호출 실패: 406 Client Error: Not Acceptable ...", "ok":false}`.
  마스터가 보고한 대로 최소 7/16부터 지속 중. 개발팀이 원인 조사 중이므로 여기서는
  건드리지 않음.
- **반면 생필품(참가격) 쪽은 정상.** `/api/consumer-prices?fast=1` 직접 호출 결과
  `count: 604`, `inspect_day: 20260626` 정상 응답. `.github/workflows/refresh-consumer-prices.yml`
  이 매일 06:00 KST에 스냅샷을 갱신하며 KAMIS와 무관한 별도 소스(한국소비자원 참가격)라
  이번 장애의 영향을 받지 않음.
- **자동화 코드에서 발견한 추가 위험(기획팀 관점에서 컨텐츠 캘린더에 직접 영향)**:
  `generate_blog_post.py` `main()` 함수 L921-926를 보면
  ```python
  try:
      items = fetch_today_prices()
      enriched = pick_candidates(items, n=TOP_N, seasonal_keywords=produce_keywords or None)
  except Exception as e:
      print(f"가격/제휴 데이터 수집 실패, 오늘은 건너뜁니다: {e}")
      return
  ```
  즉 `fetch_today_prices()`(KAMIS 호출)가 실패하면 **생필품 폴백을 시도하기도 전에
  함수 전체가 즉시 종료**됩니다. 뒤에 있는 "생필품 단독/혼합 포스트" 로직(L935~961,
  절기 이벤트가 생필품과 연관되면 우선 반영하는 로직 포함)은 이 지점에서 이미 return된
  뒤라 절대 실행되지 않습니다. 즉 **KAMIS가 복구되지 않은 채로 다음 자동 발행일
  (7/21, 7/24)을 맞으면, 참가격 데이터가 멀쩡함에도 불구하고 자동화가 아예 글을
  하나도 발행하지 못하고 조용히 건너뛸 가능성이 높습니다.** 중복(7/25)이라는 좋은
  절기 소재(consumer_keywords: 삼계탕/홍삼/영양제)가 있는데도 코드 실행 순서 때문에
  활용되지 못하는 상황.
  → 이건 콘텐츠 기획 범위를 벗어난 코드 수정 사항이라 별도로 개발팀에 태스크를
  플래그해두었습니다(세션 내 spawn_task). 기획팀 차원에서는 **KAMIS 복구 전까지
  발행 공백이 실제로 발생하는지 7/21 발행 직후 반드시 확인**하고, 비어있으면 수동
  개입(예: `python generate_blog_post.py`를 로컬에서 today-prices 우회하도록 임시
  수정해 생필품 단독 글 강제 발행, 또는 개발팀 코드 수정 완료 후 workflow_dispatch로
  수동 재실행)이 필요합니다.

## 4. 우선순위별 콘텐츠 제안 (GA4 근거 기반)

숫자 근거가 있는 순서로 우선순위를 매김. KAMIS 장애가 지속되는 동안은 **생필품(참가격)
비중을 의도적으로 높이는 방향**을 전제로 함(현재 자동화 확률은 생필품 단독 20%,
혼합 30%, 순수 농산물 50%인데, KAMIS 복구 전까지는 사실상 농산물 100%가 실패해
아무것도 안 나갈 위험이 있으므로).

### 우선순위 1 — 중복(7/25) 생필품 각도로 즉시 확보
- 근거: SEASONAL_EVENTS에 이미 중복의 `consumer_keywords: ["삼계탕", "홍삼", "영양제"]`가
  정의돼 있고, 참가격 API는 정상 작동 확인됨. 3번 항목의 코드 버그 때문에 자동화가
  이 경로를 못 탈 위험이 크므로, 7/21 발행 실패 시 수동으로라도 "홍삼/영양제 최저가"
  생필품 글을 채워야 함. 초복 글(체류시간 6.6초)보다는 "농산물 등락률형" 후킹을
  가미해 체류시간을 개선해볼 것(예: 참가격 대비 절약폭을 숫자로 크게 제시).

### 우선순위 2 — 7/26~8/8 공백 구간: "폭염/여름휴가" 생필품 시리즈
- 근거: 2번 항목에서 확인된 명확한 캘린더 공백(발행 4회분). 동시에 이 구간은
  KAMIS 장애 지속 가능성이 있는 구간과 겹치므로, 애초에 농산물 의존도가 낮은
  소재를 선택하는 것이 이중으로 안전. `/consumer` 페이지가 최근 30일 20뷰/7명으로
  `/blog` 다음으로 트래픽이 꾸준한 점도 생필품 콘텐츠에 대한 실수요가 있음을 뒷받침.
- 제안 소재(우선순위 순): 여름 위생용품(모기/벌레퇴치, 자외선차단제), 휴가철 용품
  (아이스박스, 휴대용 선풍기), 장마 직후 습기/곰팡이 마무리 특집(제습제 후속편).

### 우선순위 3 — 말복(8/14) 준비
- 근거: 8/11, 8/14 발행일 모두 말복 활성 구간(8/9~14)에 걸림. 이미 캘린더에 반영돼
  있어 별도 조치 불필요하나, 초복/중복과 소재가 겹치므로(닭/마늘/전복/인삼 동일
  키워드) 3번째 반복이 되는 시점 — 제목/후킹 문구를 초복·중복과 겹치지 않게 다양화
  필요(TITLE_TEMPLATES는 랜덤 선택이라 자동으로 어느 정도 분산되지만, 수동 검수 시
  같은 표현 반복 여부 확인 권장).

### 우선순위 4 — 카테고리 다양성 확보 (생필품)
- 근거: 지금까지 발행된 생필품 글은 장마 특집(생활용품·세제) 1건뿐. `CONSUMER_CAT_MAP`에
  정의된 13개 카테고리 중 신선식품/수산물/가공식품/유제품육가공/조미료양념/과자빙과/
  음료주류/위생바디케어/의약외품뷰티/반려동물 등 대부분이 아직 다뤄지지 않음.
  절기와 무관하게, 공백 구간(7/28~8/7) 중 최소 1~2건은 절기 연계 없이 순수
  "카테고리 다양성" 목적으로 생필품 랜덤 특집(예: 반려동물 용품, 유제품)을 넣어
  포트폴리오를 넓히는 것을 제안. 표본이 작아 아직 어떤 카테고리가 잘 되는지 데이터가
  없으므로, 다양화 자체가 다음 GA4 분석의 근거를 쌓는 목적도 겸함.

### 우선순위 5 (관찰 과제, 즉시 실행 아님) — Organic Search 유입 0건 모니터링
- 근거: 1번 섹션 참고. 8월 초까지도 Organic Search가 계속 0이면, 콘텐츠 소재
  문제가 아니라 색인/SEO 기술 문제일 가능성이 있으므로 개발팀에 사이트맵 제출
  여부·Search Console 등록 여부 확인 요청을 검토.
